- При проксировании DNS с помощью XKeen, в профиле "Политика по умолчанию" отсутствует интернет, создайте пользовательсую политкику вместо этого профиля
- Built-in политика Keenetic «Без доступа в интернет» блокируется через ipset `xkeen_deny_mac` (hash:mac) с RETURN-правилом в PREROUTING до xkeen-перехвата. Синхронизация MAC-ов с hotspot API выполняется при `xkeen -start`, на каждой netfilter.d-инвокации (`/opt/etc/ndm/netfilter.d/proxy.sh`) и на schedule.d-событиях (`/opt/etc/ndm/schedule.d/00-xkeen-hotspot-sync.sh`), что даёт динамику без `xkeen -restart` при работе расписаний родительского контроля. **Ограничение:** `-m mac --mac-source` видит L2-MAC только для устройств в одном broadcast-домене с роутером (LAN-bridge, Wi-Fi, guest-segment); за L3-VLAN с маршрутизацией MAC-источника не виден, и фикс безвредно неактивен — для таких сегментов используйте пользовательскую политику.
