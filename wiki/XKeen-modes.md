## Режимы работы XKeen

**TProxy**

Полностью прозрачный режим на уровне ядра. Перехватывает и UDP-, и TCP-трафик без изменения исходного IP-адреса клиента. Лучший выбор для полной прозрачности и совместимости с играми/видеозвонками (UDP).

**Redirect**

Классический режим прозрачного прокси через REDIRECT в iptables. Работает только с TCP. UDP не поддерживается. Подходит для старых ядер или когда UDP не нужен.

**Hybrid**

Комбинированный режим:

- UDP → TProxy

- TCP → Redirect

UDP работает через TProxy, TCP — через более простой и стабильный Redirect. Рекомендуется для сценариев, когда TProxy для TCP вызывает проблемы.

**Other**

Не является режимом прозрачного проксирования.
В этом режиме XKeen не создаёт правила iptables и не перехватывает трафик автоматически. Вы сами настраиваете маршрутизацию, например, поднимаете Socks5-прокси на роутере и вручную направляете через него клиентов.

---

Рекомендуемыми режимами для XKeen являются **TProxy** и **Hybrid**

### О необходимости освобождения 443 порта

При использовании режима чистого TProxy с любым ядром проксирования требуется освободить 443 порт управления роутером в веб-интерфейсе на странице "Пользователи и доступ". Вопреки опасениям пользователей, данный TCP порт наружу не открывается, а используется исключительно для внутренних нужд модуля `xt_TPROXY.ko` в KeeneticOS ([подробнее на форуме](https://forum.keenetic.ru/topic/17122-не-работает-tproxy-на-ipv4/#findComment-174174)). В результате освобождения 443 порта и переноса его на другой, меняются ссылки доступа к роутеру из интернета для всех сервисов (e.g. SSTP-сервер). Например, было https://myrouter.netcraze.link, станет https://myrouter.netcraze.link:8443 (после двоеточия укажите порт, который выбрали на замену 443). Для остальных режимов XKeen (Hybrid, Redirect, Other) перенос 443 порта не требуется.
