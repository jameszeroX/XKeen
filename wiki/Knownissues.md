- При проксировании DNS с помощью XKeen, во встроенном профиле Кинетика "Политика по умолчанию" отсутствует интернет. Решение - создайте и используйте пользовательскую политику, вместо "Политики по умолчанию" [Подробнее](https://github.com/jameszeroX/XKeen/wiki/Configuration#проксирование-dns)

- При использовании NextDNS или нескольких DNS-профилей в Кинетике, при запрете транзита DNS у части клиентов пропадает интернет. Решение — разрешить транзит DNS в роутере. См. [issue #19](https://github.com/jameszeroX/XKeen/issues/19)

- Проксирование трафика клиентов SSTP-сервера KeeneticOS не работает через XKeen, когда он в режиме TProxy. Решение - используйте режим Hybryd, это известная особеность реализации SSTP у Keenetic

- Проксирование может не работать для удаленных клиентов, подключённых к роутеру по VPN (e.g. WireGuard) или при подключении роутера через мобильный интернет. Решение - отключите проксирование Entware (`xkeen -pr off`) и GeoIPSET (`xkeen -dgips`)

- При подключении к роутеру нескольких провайдеров, XKeen работает через "Основное подключение", даже если в политике XKeen отметить галкой "Резервное". Исправлено в XKeen 2.0.1 - [Подробнее](https://github.com/jameszeroX/XKeen/wiki/FAQ#22-два-провайдера--подключение-через-один) 

- При установке Mihomo предлагаются последние 8 версий, включая `Prerelease-Alpha`, которая недоступна для загрузки. Решение - выбирайте версии с цифрами в версии. Исправлено в XKeen 2.0.1

- При не установленных геофайлах обновление GeoIPSET командой `xkeen -ug` завершается неудачей. Решение: установите любой геофайл, даже если не пользуетесь им. Исправлено в XKeen 2.0.1

- При неудачной загрузке геофайлов происходит сбой проксирования. Исправлено в XKeen 2.0.1
