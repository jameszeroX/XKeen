- При проксировании DNS с помощью XKeen, во встроенном профиле Кинетика "Политика по умолчанию" отсутствует интернет. Решение - создайте и используйте пользовательскую политику, вместо "Политики по умолчанию"
- При подключении к роутеру нескольких провайдеров, XKeen работает через "Основное подключение", даже если в политике XKeen отметить галкой "Резервное" - [Решение](https://github.com/jameszeroX/XKeen/wiki/FAQ#22-два-провайдера--подключение-через-один)
- При использовании NextDNS или нескольких DNS-профилей в Кинетике, при запрете транзита DNS у части клиентов пропадает интернет. Решение — разрешить транзит DNS в роутере. См. [issue #19](https://github.com/jameszeroX/XKeen/issues/19)
- На некоторых архитектурах (`mipsle`) ранее наблюдались проблемы с конфигурацией Mihomo из-за бага в `yq 4.52.4` ([yq#2609](https://github.com/mikefarah/yq/issues/2609)). С версии XKeen 2.0 Beta workaround отключён — используется официальный дистрибутив yq. При повторении проблемы установите в `01_info_variable.sh` `yq_use_workaround="true"`
