## Конфигурационный файл xkeen.json

При чистой установке XKeen пустой файл настроек создаётся по пути `/opt/etc/xkeen/xkeen.json`. При необходимости можете заполнить его по следующему примеру, соблюдая синтаксис json. Можете заранее перед установкой XKeen создать этот файл, в этом случае настройки повторных загрузок или пользовательcкого прокси будут применены на этапе установки XKeen. Комментарии в файле xkeen.json не допустимы.

```
{
  "xkeen": {
    "gh_proxy": "",
    "retries_download": "",
    "retry_delay_download": "",
    "policy": [
      {
        "name": "xkeen0",
        "port": ""
      },
      {
        "name": "xkeen1",
        "port": "80,443,596:599,1400,3478,5222"
      },
      {
        "name": "xkeen2",
        "port": "!7777,8888:9999"
      }
    ]
  }
}
```

- `gh_proxy` - пользовательский прокси для загрузок с GitHub (пример `"gh_proxy": "https://ghproxy.me",`)
- `retries_download` - количество повторных попыток загрузки с GitHub (пример `"retries_download": "3",`)
- `retry_delay_download` - количество секунд между попытками повторных загрузок (пример `"retry_delay_download": "10",`)
- `xkeen0` - пользовательская политика, проксирование на всех портах (имя политики произвольное, важно соблюдение json-структуры)
- `xkeen1` - пользовательская политика, проксирование на перечисленных портах (имя политики произвольное, важно соблюдение json-структуры)
- `xkeen2` - пользовательская политика, проксирование на всех портах, кроме перечисленных (имя политики произвольное, важно соблюдение json-структуры)
