## Конфигурационный файл xkeen.json

При чистой установке XKeen создаётся пустой файл настроек по пути /opt/etc/xkeen/xkeen.json. При необходимости вы можете заполнить его по примеру ниже, соблюдая синтаксис JSON. Вы также можете создать этот файл заранее — до установки XKeen. В этом случае указанные в нём настройки (например, параметры повторных загрузок или пользовательский прокси) будут применены уже на этапе установки.

```
{
  "xkeen": {
    "gh_proxy": "",
    "retries_download": "",
    "retry_delay_download": "",
    "policy": [
      {
        "name": "xkeen0",
        "port": ""
      },
      {
        "name": "xkeen1",
        "port": "80,443,596:599,1400,3478,5222"
      },
      {
        "name": "xkeen2",
        "port": "!7777,8888:9999"
      }
    ],
    "geodata": [
      {
        "file": "geosite_name.dat",
        "url": "https://site.com/geosite.dat"
      },
      {
        "file": "geoip_name.dat",
        "url": "https://site.com/geoip.dat"
      }
    ]
  }
}
```

- `gh_proxy` - пользовательский прокси для загрузок с GitHub (пример `"gh_proxy": "https://ghproxy.me",`)
- `retries_download` - количество повторных попыток загрузки с GitHub (пример `"retries_download": "3",`)
- `retry_delay_download` - количество секунд между попытками повторных загрузок (пример `"retry_delay_download": "10",`)
- `policy` - раздел, описывающий пользовательские политики маршрутизации
- `xkeen0` - пользовательская политика, проксирование на всех портах (имя политики произвольное)
- `xkeen1` - пользовательская политика, проксирование на перечисленных портах (имя политики произвольное)
- `xkeen2` - пользовательская политика, проксирование на всех портах, кроме перечисленных (имя политики произвольное)
- `geodata` - раздел, описывающий пользовательские геофайлы, которые требуется обновлять средствами XKeen
- `file` - имя геофайла в директории `/opt/etc/xray/dat/`
- `url` - ссылка для загрузки геофайла
