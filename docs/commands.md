# Справочник флагов `xkeen`

Полный список флагов диспетчера [`scripts/xkeen`](../scripts/xkeen). Извлечён из `help_xkeen()` в [`scripts/_xkeen/about.sh`](../scripts/_xkeen/about.sh). Деструктивные флаги отмечены ⚠ — они интерактивные, требуют подтверждения, не имеют опции «тихого» режима.

## Установка

| Флаг | Действие |
| --- | --- |
| `-i`, `-install` | Полный цикл: XKeen + Xray + Mihomo + GeoFile/GeoIPSET |
| `-io` | OffLine-установка XKeen из локальной флешки |
| `-toff` | Отключить таймаут `curl` для медленных каналов: `xkeen -i -toff` |
| `-health` | Базовая проверка исправности Entware перед установкой XKeen |

## Балансировка по фактической скорости

| Флаг | Действие |
| --- | --- |
| `-sb` | Открыть меню балансировки (статус, включить/выключить, замер сейчас) |
| `-sb on` | Включить балансировку по скорости |
| `-sb off` | Выключить и снять override балансировщика |
| `-sb status` | Показать состояние, текущую ноду и последние события |
| `-sbt` | Один цикл замера и переключения — служебный флаг для cron, вручную не нужен |

Штатная балансировка Xray выбирает ноду по задержке (`leastPing`). `-sb` добавляет выбор по **фактической скорости**: по расписанию замеряет каждую ноду балансировщика и переставляет выбор на самую быструю, если она обгоняет текущую больше чем на гистерезис.

Фича спит, пока не включена (`.xkeen.speed_balancer.enabled` в `xkeen.json`); без флага XKeen работает как раньше. При первом включении, если gRPC api Xray не настроен, `-sb on` предложит добавить его автоматически (с подтверждением). Параметры настраиваются в `xkeen.json`:

| Ключ (`.xkeen.speed_balancer.*`) | По умолчанию | Смысл |
| --- | --- | --- |
| `enabled` | `false` | Включена ли балансировка |
| `interval` | `15` | Период замера, минут |
| `hysteresis` | `25` | На сколько процентов кандидат должен обгонять текущую, чтобы переключиться |
| `balancer` | `balancer` | Тег балансировщика в файле роутинга |
| `max_time` | `8` | Таймаут одного замера, секунд |
| `test_url` | Cloudflare `__down` | URL для замера скорости |
| `routing_file` | `05_routing.json` | Имя файла роутинга в каталоге конфигов Xray (для нестандартной раскладки) |
| `outbounds_file` | `04_outbounds.json` | Имя файла с outbound'ами в каталоге конфигов Xray |

Требует блок `api` (RoutingService) и probe-inbound в конфигурации Xray — их добавляет автонастройка. Замер идёт через отдельный probe-канал и боевой трафик не затрагивает. Механизм опирается на `RoutingService.OverrideBalancerTarget` — override без TTL, поэтому `-sb off` обязательно снимает его, а после перезапуска Xray выбор возвращается к `leastPing` до следующего замера.

**Ограничение.** Модуль рассчитан на балансировщик с пулом взаимозаменяемых узлов (стандартный `sub-`-пул): он ранжирует узлы одного балансировщика, заданного тегом `balancer`, и переставляет выбор на быстрейший. Схема с несколькими балансировщиками одновременно — в частности каскадная отказоустойчивость из [третьей части](https://jameszero.net/4576.htm) — не поддерживается: там у балансировщика одна ветка (ранжировать нечего), а принудительный выбор мешал бы failover. С дефолтным тегом на такой конфигурации модуль ничего не делает. Если вы вручную перегенерируете роутинг (например меняя список outbound'ов), выполните `xkeen -sb on` повторно, чтобы вернуть правило `api`.

## Переменные окружения

| Переменная | Значение | Эффект |
| --- | --- | --- |
| `XKEEN_FOREGROUND` | `1` | Отключает self-detach при запуске без TTY. Использовать в скриптах с синхронной семантикой (`xkeen -start && cleanup`) |
| `XKEEN_DETACHED` | `1` | Внутренний маркер — выставляется самим `xkeen` после форка через `start-stop-daemon -b`. Руками не выставлять |
